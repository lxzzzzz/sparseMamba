# 100m / 200m 完整命令链

这份文件把两套流程都写全了：

1. 训练检测器
2. 生成 tracking infos
3. 生成 detector cache
4. 训练 tracker
5. 评测 tracker

默认工程根目录：

```bash
cd /home/lx/sparseMamba
```

如果你在 AutoDL 上跑，通常根目录会变成：

```bash
cd /root/autodl-tmp/sparseMamba
```

## 0. 先说明一件事

`tracking_infos_train.pkl` 和 `tracking_infos_val.pkl` 只需要按你的 tracking 数据集生成一次。

只要下面这些没变，就不用重复生成：

- tracking 数据目录结构
- 序列划分
- 标注
- `create_tracking_infos.py` 逻辑

但是：

- 100m detector 和 200m detector 要分别训练
- 100m cache 和 200m cache 要分别生成
- 100m tracker 和 200m tracker 也要分别训练

## 1. 先准备 tracking infos

这一步一般只做一次。

```bash
python tools/create_tracking_infos.py \
  --data_path /your/tracking/data/root \
  --save_path /your/tracking/data/root \
  --splits train val
```

会生成：

- `tracking_infos_train.pkl`
- `tracking_infos_val.pkl`

如果你在 AutoDL 上，推荐写成：

```bash
python tools/create_tracking_infos.py \
  --data_path /root/autodl-tmp/data/dair_v2x_tracking \
  --save_path /root/autodl-tmp/data/dair_v2x_tracking \
  --splits train val
```

## 2. 100m 全流程

### 2.1 训练 100m 检测器

```bash
python tools/train.py \
  --cfg_file tools/cfgs/dair_v2x_models/fusion_voxelnext_100m.yaml \
  --extra_tag fusion_det_100m \
  --workers 4
```

检测训练输出目录通常会是：

```text
output/dair_v2x_models/fusion_voxelnext_100m/fusion_det_100m/
```

常用 checkpoint 位置：

```text
output/dair_v2x_models/fusion_voxelnext_100m/fusion_det_100m/ckpt/checkpoint_epoch_30.pth
```

### 2.2 生成 100m detector cache

如果你沿用配置里的默认路径：

```bash
python tools/generate_tracking_cache.py \
  --detector_cfg tools/cfgs/dair_v2x_models/fusion_voxelnext_100m.yaml \
  --data_cfg tools/cfgs/dataset_configs/dair_v2x_tracking_dataset_100m.yaml \
  --ckpt output/dair_v2x_models/fusion_voxelnext_100m/fusion_det_100m/ckpt/checkpoint_epoch_30.pth \
  --save_dir /your/tracking/data/root/cache/fusion_voxelnext_100m \
  --batch_size 1 \
  --workers 2
```

如果你在 AutoDL 上，建议直接写：

```bash
python tools/generate_tracking_cache.py \
  --detector_cfg tools/cfgs/dair_v2x_models/fusion_voxelnext_100m.yaml \
  --data_cfg tools/cfgs/dataset_configs/dair_v2x_tracking_dataset_100m.yaml \
  --ckpt output/dair_v2x_models/fusion_voxelnext_100m/fusion_det_100m/ckpt/checkpoint_epoch_30.pth \
  --save_dir /root/autodl-tmp/cache/fusion_voxelnext_100m \
  --batch_size 1 \
  --workers 2
```

### 2.3 训练 100m tracker

如果 `track_mamba_100m.yaml` 里的默认路径就是你实际使用的路径，可以直接跑：

```bash
python tools/train_tracker.py \
  --cfg_file tools/cfgs/tracking_models/track_mamba_100m.yaml \
  --extra_tag track_mamba_100m \
  --workers 4
```

如果你在 AutoDL 上，通常需要覆盖路径：

```bash
python tools/train_tracker.py \
  --cfg_file tools/cfgs/tracking_models/track_mamba_100m.yaml \
  --extra_tag track_mamba_100m \
  --workers 4 \
  --set \
  DATA_CONFIG.ROOT_DIR /root/autodl-tmp/data/dair_v2x_tracking \
  DATA_CONFIG.CACHE_DIR /root/autodl-tmp/cache/fusion_voxelnext_100m
```

### 2.4 评测 100m tracker

```bash
python tools/eval_tracker.py \
  --cfg_file tools/cfgs/tracking_models/track_mamba_100m.yaml \
  --ckpt output/cfgs/tracking_models/track_mamba_100m/track_mamba_100m/ckpt/best_model.pth \
  --extra_tag track_mamba_100m \
  --set \
  DATA_CONFIG.ROOT_DIR /root/autodl-tmp/data/dair_v2x_tracking \
  DATA_CONFIG.CACHE_DIR /root/autodl-tmp/cache/fusion_voxelnext_100m
```

## 4. 建议的实际执行顺序

建议先把 100m 跑通，再做 200m：

1. 生成一次 `tracking_infos`
2. 训练 100m detector
3. 生成 100m cache
4. 训练 100m tracker
5. 评测 100m tracker
6. 训练 200m detector
7. 生成 200m cache
8. 训练 200m tracker
9. 评测 200m tracker

## 6. 最容易错的地方

- `tracking_infos` 不用因为 100m/200m 重复生成，但 detector cache 必须分开生成。
- 100m 的 detector、100m 的 cache、100m 的 tracker 配置必须配套。
- 200m 的 detector、200m 的 cache、200m 的 tracker 配置必须配套。
- 不要把 100m checkpoint 拿去生成 200m cache。
- 不要把 200m cache 拿给 100m tracker 训练。
- 如果你的实际数据路径不是配置里的默认路径，就用 `--set` 覆盖。
